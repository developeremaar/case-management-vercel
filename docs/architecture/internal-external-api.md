# Internal / External API

## API داخلي
- يخدم واجهة الويب.
- يخدم تطبيق الجوال لاحقًا.
- يخدم عمليات الموظفين داخل النظام.

## API خارجي
- يخدم التكاملات.
- يخدم الجهات الشريكة.
- يخدم البوابة الخارجية.
- يدعم Webhooks.

## مبادئ api_clients / api_keys / scopes / request_logs
- `api_clients`: تعريف عميل خارجي مرتبط بجهة.
- `api_keys`: مفاتيح وصول مرتبطة بالعميل.
- `scopes`: نطاقات صلاحية دقيقة.
- `request_logs`: سجل كامل للاستخدام والأخطاء.

## ضوابط إلزامية
- لا يتم تخزين API keys كنص صريح (hash فقط).
- كل API خارجي يجب أن يرتبط بـ organization_id.
- دعم rate limits وscopes مطلوب معماريًا من الآن (وتدرجيًا في التنفيذ).
