# SaaS Architecture

النظام يعتمد نموذج **SaaS متعدد الجهات**:
- كل جهة لها بيانات مستقلة.
- المستخدم قد ينتمي لأكثر من جهة.
- الصلاحيات تعتمد على membership داخل organization.

## الطبقات المعتمدة
1. طبقة المنصة.
2. طبقة الجهة.
3. طبقة المستخدم والصلاحيات.
4. طبقة التشغيل.
5. طبقة التكاملات والإشعارات.
6. طبقة API الداخلي والخارجي.
7. طبقة البوابة الخارجية مستقبلًا.

## قواعد تنظيمية
- الفروع اختيارية.
- الأقسام أساسية داخل الجهة.
- كل جهة قد تملك مسارات عمل مختلفة.
- الاشتراكات والباقات مطلوبة من الآن.
- التكاملات مطلوبة من الآن (WhatsApp/SMS/Email/Domain/Webhooks).
- الإشعارات مطلوبة من الآن (الموظفين، المستفيدين، الحالات، الأقسام، الأدوار، وتطبيق الجوال لاحقًا).
